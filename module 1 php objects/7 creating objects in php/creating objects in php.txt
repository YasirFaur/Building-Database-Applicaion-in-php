This lesson explains the difference between old PHP4 style programming and modern Object-Oriented Programming (OOP) in PHP. In PHP4, developers often used arrays with keys and values to build data structures, for example storing a person’s full name, given name, family name, and room. Functions were then written to handle the variations, such as printing either a full name or combining given and family names. However, repeating this logic many times becomes inefficient.

With OOP, PHP introduces the class keyword as a template for objects. A class defines attributes, such as $fullname, $givenname, $familyname, and $room, and also methods, such as get_name(). Each object created from the class holds its own data, but shares the same methods. The special variable $this allows a method to refer to the specific object’s data. For example, $chuck and $colleen are two objects of the class Person, each with their own values. Calling get_name() returns the correct name depending on which object is used. This shows how OOP organizes data and code together, making it more reusable and structured.
